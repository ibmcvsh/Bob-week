# 💊 Pill Tracker Lab – Instructions

## 🎯 Overview

This is a lightweight demo showing how to build and improve an application using Bob AI.

In this session, you will:
- Generate a working pill tracker app from a specification
- Iteratively improve the app using natural language prompts
- Learn how to effectively guide AI to build and refine software

This is a hands-on, prompt-driven exercise, not a production build.


---

## ⏱️ Session Flow (90 Minutes)

### 🟢 0:00 – 0:15 | Generate the App
- Review the spec
- Prompt Bob to build the application
- Validate that core features are working

### 🔵 0:15 – 1:30 | Improve with Prompts
- Add features
- Refine functionality
- Improve usability
- Learn how to iterate effectively with AI


---

## 🚀 Quick Start

1. Open Bob  
2. Open the pill-tracker directory 
3. Run this starter prompt:

        Build a pill tracker application based on the specifications in pill-tracker-spec.md.
        Create a simple single-page app using HTML, CSS, and JavaScript.

4. Review the generated app  
5. Begin enhancements using the exercises below  


---

## 🧪 Exercises & Enhancements

### Core Improvements

#### Add reminder notifications when pills are due:

    Add a notification system that alerts users when a pill is due.  
    Show a visual indicator with the medication name and time.


#### Improve validation (e.g., prevent duplicate “taken” actions):

    Prevent users from marking a pill as taken more than once within its scheduled time.  
    Show a clear error message if they try.


#### Enhance the UI (colors, layout, clarity):

    Improve the design:
    - Color code pill status (due, taken, upcoming)
    - Improve layout and spacing
    - Add simple icons


#### Improve time simulation (more controls, better visibility):

    Add buttons to advance time by 1 hour, 4 hours, and 8 hours.  
    Highlight pills as they become due.

## 🛠️ Troubleshooting & Tips

### ❗ If the output isn’t right
- Be more specific in your prompt  
- Reference exactly what to change  
- Break the request into smaller steps  

### 🐛 If something breaks
Debug this issue and explain what went wrong.

### 🧠 If the code is confusing
Explain this step-by-step in plain English.

## ✅ Best Practices

- Work iteratively
